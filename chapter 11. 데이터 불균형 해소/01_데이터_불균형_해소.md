# 📘 Chapter 11. 데이터 불균형 해소

> **학습 목표**
> - 데이터 불균형(Class Imbalance)이 모델 학습에 미치는 영향을 이해한다.
> - 불균형 해소를 위한 주요 기법들의 접근 방법, 세부 내용, 기대효과를 파악한다.
> - 실무에서 상황에 맞는 적절한 기법을 선택할 수 있다.

---

## 1. 데이터 불균형이란?

**데이터 불균형(Class Imbalance)**이란 학습 데이터에서 **특정 클래스의 샘플 수가 다른 클래스에 비해 현저히 적은 상태**를 말합니다.

```
예시: 불량 검출 데이터셋

  정상(OK):  9,500장  ██████████████████████████████████████████████████  95%
  불량(NG):    500장  ███                                                 5%
```

### 📌 왜 문제가 되는가?

| 문제 | 설명 |
|------|------|
| **다수 클래스 편향** | 모델이 "전부 정상"이라고 예측해도 95% 정확도 → 실제로는 쓸모없는 모델 |
| **소수 클래스 학습 부족** | 불량 데이터를 거의 보지 못해 불량 패턴을 제대로 학습하지 못함 |
| **Loss 기여도 불균형** | 다수 클래스의 Loss가 전체 Loss를 지배하여 소수 클래스 방향으로 학습이 잘 안 됨 |
| **평가 지표 왜곡** | Accuracy만으로는 성능 판단이 불가 → F1-Score, AUROC 등을 사용해야 함 |

---

## 2. 불균형 해소 기법 분류 체계

데이터 불균형 해소 기법은 크게 **4가지 레벨**로 나뉩니다.

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                          데이터 불균형 해소 기법                                  │
├──────────────────┬──────────────────┬─────────────────────┬────────────────────┤
│  ① 데이터 레벨    │  ② 샘플링 레벨    │  ③ 손실 함수 레벨    │  ④ 패러다임 전환    │
│                  │                  │                     │                    │
│  • Augmentation  │  • Sampler       │  • Weighted CE      │  • Anomaly         │
│   (기본 증강)     │   (Weighted      │    (클래스 가중치)    │    Detection       │
│  • Mixup/CutMix  │    Random        │  • Focal Loss       │    (정상만 학습)     │
│   (고급 증강)     │    Sampler)      │    (샘플 난이도)     │                    │
└──────────────────┴──────────────────┴─────────────────────┴────────────────────┘

  ①②③: 정상 + 불량 모두 학습하되, 불균형을 보정하는 접근
  ④  : 발상을 전환하여, 정상만 학습하고 "정상과 다른 것"을 불량으로 판단
```

---

## 3. ① 데이터 레벨: Data Augmentation ⭐

> **접근 방법**: 소수 클래스의 원본 데이터를 다양한 변환을 통해 **새로운 학습 데이터로 확장**하여 불균형을 완화한다.

---

### 3-1. 기본 증강 (Geometric / Color 변환)

| 기법 | 설명 | 예시 |
|------|------|------|
| **회전 (Rotation)** | 이미지를 일정 각도로 회전 | ±15°, ±30° |
| **반전 (Flip)** | 좌우/상하 반전 | Horizontal Flip, Vertical Flip |
| **크롭 (Random Crop)** | 이미지의 일부 영역을 무작위 잘라냄 | 224×224 → 200×200 영역 추출 |
| **색상 변환 (Color Jitter)** | 밝기, 대비, 채도, 색조를 랜덤 조절 | brightness=0.3, contrast=0.3 |
| **노이즈 추가 (Noise)** | 가우시안 노이즈를 추가하여 변형 | σ = 0.01 ~ 0.05 |

---

### 3-2. 고급 증강 (Advanced)

| 기법 | 설명 |
|------|------|
| **Mixup** | 두 이미지를 **선형 보간(linear interpolation)**으로 합성하고, 라벨도 비율에 맞게 혼합 |
| **CutMix** | 한 이미지의 일부 영역을 잘라내고, 다른 이미지의 해당 영역을 붙여넣기 |
| **CutOut (Erasing)** | 이미지의 일부 영역을 랜덤하게 마스킹(0으로 채움) |
| **GAN 기반 생성** | GAN으로 소수 클래스의 **합성 이미지를 생성** (학습 데이터가 매우 적을 때 유용) |

**Mixup / CutMix 핵심 원리**

```
Mixup:
  새 이미지  = λ × 이미지A  +  (1-λ) × 이미지B
  새 라벨    = λ × 라벨A    +  (1-λ) × 라벨B

  예: λ=0.7이면, 이미지A를 70%, 이미지B를 30% 비율로 합성

CutMix:
  새 이미지 = 이미지A의 배경 + 이미지B에서 잘라낸 패치
  새 라벨   = 패치 면적 비율에 따라 라벨을 혼합

  예: 패치가 전체의 40%이면, 라벨A:60% + 라벨B:40%
```

### 적용 시 기대효과

- ✅ **데이터 다양성 증가**: 원본 데이터를 변형하여 모델이 더 다양한 패턴을 학습
- ✅ **과적합 방지**: 동일한 이미지를 반복 학습하지 않아 일반화 성능 향상
- ✅ **소수 클래스 강화**: 소수 클래스에 집중적으로 적용하면 데이터 분포 균형화
- ✅ **정보 손실 없음**: 기존 데이터를 삭제하지 않으므로 학습 정보가 보존됨

### 주의사항
- 증강은 **학습(Train) 데이터에만** 적용해야 함 (검증/테스트 데이터에는 적용 ❌)
- 도메인 특성을 고려해야 함 (예: 숫자 "6"을 180° 회전하면 "9"가 됨)

## 4. ② 샘플링 레벨: Sampler ⭐

> **접근 방법**: 데이터 자체를 변경하지 않고, DataLoader가 **Batch를 구성하는 방식**을 조정하여 소수 클래스를 더 자주 학습하도록 한다.

---

### 4-1. PyTorch Sampler 체계

PyTorch의 `DataLoader`는 **Sampler**를 통해 데이터를 뽑는 순서를 제어합니다.
Sampler는 "어떤 순서로 데이터를 꺼낼 것인가?"를 결정하는 역할을 합니다.

```
DataLoader 동작 흐름:

  Dataset (전체 데이터)
      │
      ▼
  Sampler (데이터를 뽑는 순서/확률을 결정)
      │
      ▼
  BatchSampler (Sampler에서 뽑은 인덱스를 Batch 단위로 묶음)
      │
      ▼
  collate_fn (Batch 내 샘플들을 하나의 Tensor로 합침)
      │
      ▼
  최종 Batch 출력
```

#### PyTorch 제공 Sampler 종류

| Sampler | 설명 | 용도 |
|---------|------|------|
| **SequentialSampler** | 인덱스를 **순서대로** (0, 1, 2, ...) 반환 | 평가/테스트 시 순서 유지 |
| **RandomSampler** | 인덱스를 **무작위** 순서로 반환 | 일반 학습 (`shuffle=True` 시 자동 사용) |
| **SubsetRandomSampler** | 지정한 **인덱스 목록** 내에서 무작위 반환 | Train/Val 분리 시 활용 |
| **WeightedRandomSampler** | 각 샘플에 **가중치(확률)**를 부여하여 추출 | ⭐ **불균형 해소 핵심** |
| **BatchSampler** | 다른 Sampler를 감싸서 Batch 단위로 묶어줌 | 커스텀 Batch 구성 |
| **DistributedSampler** | 멀티 GPU 학습 시 데이터를 GPU별로 분배 | 분산 학습 환경 |

> 📌 `shuffle=True`로 설정하면 내부적으로 `RandomSampler`가 자동 사용됩니다.
> 커스텀 `sampler`를 지정하면 `shuffle` 옵션은 사용할 수 없습니다.

---

### 4-2. WeightedRandomSampler (불균형 해소 핵심)

**접근 방법**: 각 샘플에 가중치를 부여하여, **소수 클래스 샘플이 더 높은 확률로 선택**되도록 한다.

#### 동작 원리

```
데이터셋 분포:  정상(OK) 9,500장  |  불량(NG) 500장

일반 DataLoader (RandomSampler):
  Batch에 정상:불량 ≈ 95:5 비율로 구성 → 불량을 거의 못 봄

WeightedRandomSampler 적용:
  각 샘플에 가중치 부여
    정상 샘플 가중치 = 1/9500 ≈ 0.000105
    불량 샘플 가중치 = 1/500  ≈ 0.002000  (약 19배 높음)

  → Batch에 정상:불량 ≈ 50:50에 가까운 비율로 구성
  → 불량 샘플을 자주 학습하게 됨
```

#### PyTorch 구현 예시

```python
from torch.utils.data import DataLoader, WeightedRandomSampler

# 각 클래스의 샘플 수
class_counts = [9500, 500]  # [정상, 불량]

# 각 클래스의 가중치 = 1 / 클래스 샘플 수
class_weights = [1.0 / c for c in class_counts]

# 각 샘플별 가중치 할당 (labels: 전체 데이터의 라벨 리스트)
sample_weights = [class_weights[label] for label in labels]

# WeightedRandomSampler 생성
sampler = WeightedRandomSampler(
    weights=sample_weights,
    num_samples=len(sample_weights),  # 1 Epoch에 뽑을 총 샘플 수
    replacement=True                  # 복원 추출 (같은 샘플이 여러번 뽑힐 수 있음)
)

# DataLoader에 sampler 적용
train_loader = DataLoader(dataset, batch_size=64, sampler=sampler)
# ※ sampler를 사용하면 shuffle=True를 사용할 수 없습니다
```

#### 적용 시 기대효과

- ✅ **데이터를 변형하지 않음**: 원본 데이터를 그대로 유지하면서 균형 학습 가능
- ✅ **구현이 간단**: DataLoader 옵션만 변경하면 되므로 기존 코드 수정 최소화
- ✅ **Epoch마다 다른 구성**: 매 Epoch마다 다른 조합의 Batch가 만들어져 과적합 방지
- ✅ **Augmentation과 함께 사용 가능**: 데이터 증강과 결합하면 시너지 효과

#### 주의사항
- `replacement=True`이므로 소수 클래스 샘플이 **중복 추출**될 수 있음 → Augmentation과 함께 사용 권장
- `sampler`를 사용하면 `shuffle=True` 옵션과 **동시에 사용 불가** (sampler가 순서를 제어)

## 5. ③ 손실 함수 레벨 ⭐

> **접근 방법**: 데이터와 샘플링 전략은 변경하지 않고, **Loss Function 자체를 조정하여** 소수 클래스의 학습을 강화한다.

이 레벨에는 크게 두 가지 접근법이 있습니다:

```
                        Loss Function 조정
                              │
              ┌───────────────┴───────────────┐
              │                               │
    Weighted Cross-Entropy              Focal Loss
    ─────────────────────              ──────────
    "클래스 빈도" 기반 보정              "샘플 난이도" 기반 보정
    (정적 가중치)                       (동적 가중치)
              │                               │
    소수 클래스 전체에                   어려운 개별 샘플에
    균일하게 높은 가중치                 자동으로 높은 가중치
```

---

### 5-1. Weighted Cross-Entropy (클래스 가중치) ⭐

**접근 방법**: Cross-Entropy Loss 계산 시 **클래스별 가중치(`weight`)를 부여**하여, 소수 클래스의 오분류가 전체 Loss에 더 크게 반영되도록 한다.

#### 핵심 개념

**일반 Cross-Entropy**는 모든 샘플의 Loss를 동일하게 합산합니다.
**Weighted Cross-Entropy**는 각 클래스에 **빈도 역수 비례 가중치**를 곱합니다.

```
일반 CE Loss:
  L = -log(p_정답)                    ← 모든 샘플 동일 취급

Weighted CE Loss:
  L = -w_클래스 × log(p_정답)          ← 클래스별 가중치를 곱함
```

#### 가중치 계산 방법

```
w_i = 전체 샘플 수 / (클래스 수 × 해당 클래스 샘플 수)

예시:
  전체 = 10,000장,  클래스 수 = 2
  정상(9,500장):  w_OK = 10000 / (2 × 9500) = 0.526
  불량(  500장):  w_NG = 10000 / (2 × 500)  = 10.0    ← 약 19배 높은 가중치
```

#### PyTorch 구현 예시

```python
import torch
import torch.nn as nn

# 각 클래스의 가중치 설정
class_weights = torch.tensor([0.526, 10.0]).to(device)  # [정상, 불량]

# CrossEntropyLoss에 weight 파라미터 적용
criterion = nn.CrossEntropyLoss(weight=class_weights)
```

#### 특징

| 항목 | 설명 |
|------|------|
| **조정 기준** | 클래스 빈도 (정적) — 학습 중 변하지 않음 |
| **작동 방식** | 소수 클래스 전체에 균일하게 높은 가중치 부여 |
| **구현 난이도** | ⭐ 매우 쉬움 — 코드 1~2줄 변경 |
| **적합한 상황** | 중간 수준의 불균형, 빠른 실험, 기본 베이스라인 |

#### 적용 시 기대효과

- ✅ **가장 간단한 방법**: 코드 1~2줄 변경으로 적용 가능
- ✅ **데이터 변경 불필요**: 원본 데이터를 그대로 사용
- ✅ **학습 시간 변화 없음**: 데이터 양이 증가하지 않으므로 학습 속도 유지
- ⚠️ 가중치를 너무 크게 설정하면 학습이 불안정해질 수 있음
- ⚠️ "이미 잘 맞추는 쉬운 다수 클래스 샘플"에도 동일하게 낮은 가중치 적용 → 세밀한 조절 불가

---

### 5-2. Focal Loss (샘플 난이도 기반)

**접근 방법**: 클래스 빈도가 아닌, **개별 샘플의 예측 난이도**에 따라 Loss 기여도를 동적으로 조절한다. 모델이 이미 잘 맞추는 "쉬운 샘플"의 Loss를 줄이고, 분류하기 어려운 "어려운 샘플"에 집중하도록 한다.

#### 수식

```
Focal Loss:

  FL(p_t) = -α_t × (1 - p_t)^γ × log(p_t)
                    ──────────
                    이 부분이 핵심!
                    (쉬운 샘플의 Loss를 줄이는 조절 인자)
```

| 기호 | 의미 |
|------|------|
| p_t | 모델이 정답 클래스에 대해 예측한 확률 (0~1) |
| α_t (alpha) | 클래스별 균형 가중치 — Weighted CE의 `weight`와 같은 역할 |
| γ (gamma) | **집중 파라미터** — 쉬운 샘플의 Loss를 얼마나 줄일지 제어 (일반적으로 γ = 2) |

> 📌 **Focal Loss는 Weighted Cross-Entropy를 포함하는 확장 개념**입니다.
> - `γ = 0`이면 Focal Loss = Weighted Cross-Entropy (α만 작용)
> - `γ > 0`이면 Focal Loss 고유의 "난이도 기반 조절"이 추가됨

#### 핵심 메커니즘: `(1 - p_t)^γ` 의 역할

```
모델이 정답을 확신하는 "쉬운 샘플" (p_t = 0.95):
  (1 - 0.95)² = 0.0025    ← Loss가 0.25%로 축소!  → 거의 무시

모델이 헷갈리는 "어려운 샘플" (p_t = 0.3):
  (1 - 0.3)²  = 0.49      ← Loss가 49%로 유지     → 집중 학습!

모델이 완전히 틀린 샘플 (p_t = 0.05):
  (1 - 0.05)² = 0.9025    ← Loss가 90%로 유지     → 강하게 학습!
```

#### γ 값에 따른 효과

```
γ = 0  →  Weighted Cross-Entropy와 동일 (Focal 효과 없음)
γ = 1  →  쉬운 샘플의 Loss를 약하게 감소
γ = 2  →  쉬운 샘플의 Loss를 강하게 감소 (가장 많이 사용되는 값)
γ = 5  →  쉬운 샘플의 Loss를 매우 강하게 감소
```

#### 특징

| 항목 | 설명 |
|------|------|
| **조정 기준** | 개별 샘플의 예측 확률 (동적) — 학습 중 매 Step마다 변화 |
| **작동 방식** | 쉬운 샘플 → Loss 감소, 어려운 샘플 → Loss 유지/증가 |
| **구현 난이도** | ⭐⭐ 중간 — PyTorch 기본 내장 아님 (직접 구현 또는 torchvision 사용) |
| **적합한 상황** | 극심한 불균형, 객체 탐지(배경 vs 객체), Hard Negative가 많은 경우 |

#### 적용 시 기대효과

- ✅ **어려운 샘플에 자동 집중**: γ 파라미터로 쉬운/어려운 샘플의 기여도를 자동 조절
- ✅ **Weighted CE보다 정교**: 클래스 빈도(α)뿐 아니라 개별 샘플의 난이도(γ)까지 고려
- ✅ **객체 탐지에서 특히 효과적**: RetinaNet 논문에서 제안, 배경 vs 객체 불균형에 강점
- ⚠️ PyTorch에 기본 내장되어 있지 않아 직접 구현하거나 라이브러리(torchvision) 사용 필요

---

### 5-3. 두 Loss Function 비교

| 비교 항목 | Weighted Cross-Entropy | Focal Loss |
|-----------|:----------------------:|:----------:|
| **가중치 기준** | 클래스 빈도 (정적) | 샘플별 예측 난이도 (동적) |
| **가중치 변화** | 학습 중 고정 | 매 Step마다 자동 조절 |
| **α (클래스 균형)** | `weight` 파라미터 | α 파라미터 (동일 역할) |
| **γ (난이도 조절)** | ❌ 없음 | ✅ 핵심 기능 |
| **관계** | Focal Loss의 **특수한 경우** (γ=0) | Weighted CE의 **확장 버전** |
| **구현 난이도** | ⭐ 매우 쉬움 | ⭐⭐ 중간 |
| **적합한 상황** | 중간 불균형 | 극심한 불균형 / 객체 탐지 |

## 6. ④ 패러다임 전환: Anomaly Detection 접근법

> **접근 방법**: 불량 데이터를 학습에 사용하지 않는다. **정상 데이터만 학습하여 "정상의 패턴"을 모델링**하고, 추론 시 정상 패턴에서 벗어나는 샘플을 불량으로 판단한다.

---

### 📌 언제 이 접근법을 고려하는가?

앞서 배운 ①②③ 기법들은 모두 **정상 + 불량 데이터를 함께 학습**하되, 불균형을 보정하는 방법이었습니다.
하지만 다음과 같은 상황에서는 이 접근 자체가 어렵습니다:

```
극단적 상황:

  정상(OK): 10,000장   ██████████████████████████████████████████████  99.9%
  불량(NG):     10장   ▏                                               0.1%

  → Augmentation을 해도 불량 패턴이 너무 부족
  → Sampler / Class Weights를 써도 10장으로는 불량 패턴을 학습하기 어려움
  → "불량을 학습하는 것 자체가 불가능한 수준"
```

이럴 때는 **발상을 전환**합니다:

```
기존 접근 (Classification):             새로운 접근 (Anomaly Detection):
  "정상 vs 불량 구분을 학습"                 "정상이 무엇인지만 학습"
  → 불량 데이터가 필수                      → 불량 데이터가 불필요!
  → 불량이 적으면 학습 실패                 → 정상 데이터만으로 학습 가능
                                           → "정상과 다르면 불량"으로 판단
```

---

### 6-1. 핵심 원리

```
학습 단계 (정상 데이터만 사용):
  정상 이미지들 → 모델이 "정상의 패턴"을 학습
                  (정상은 이렇게 생겼다!)

추론 단계 (새로운 이미지 입력):
  입력 이미지 → 모델이 "정상 패턴"과 비교
                 │
                 ├── 정상 패턴과 유사   → 정상 (OK) ✅
                 └── 정상 패턴과 다름   → 이상 (NG) 🚨
                      (Anomaly Score가 임계값 초과)
```

---

### 6-2. 대표적인 Anomaly Detection 기법

| 기법 | 원리 | 특징 |
|------|------|------|
| **AutoEncoder** | 정상 이미지를 압축(encode) → 복원(decode) 학습. 불량 입력은 복원이 잘 안 됨 → 복원 오차로 이상 감지 | 구현이 간단, 입문용으로 적합 |
| **PatchCore** | 정상 이미지의 패치(patch) 특징을 메모리 뱅크에 저장. 새 이미지의 패치를 메모리와 비교하여 거리가 멀면 이상 | 학습 불필요(Feature 추출만), 높은 성능 |
| **PaDiM** | 사전학습 모델(ResNet 등)의 중간 특징을 정상 이미지에서 추출 → 정상 분포를 가우시안으로 모델링 → 분포 밖이면 이상 | 패치 단위 이상 위치까지 탐지 가능 |
| **GANomaly** | GAN으로 정상 이미지 생성을 학습 → 불량 입력은 정상으로 생성하지 못함 → 입력과 생성 결과의 차이로 이상 감지 | GAN 기반, 복잡한 패턴 학습 가능 |

#### AutoEncoder 기반 Anomaly Detection 원리

```
정상 이미지 입력:
  [정상 이미지] → Encoder → Latent → Decoder → [복원 이미지]
                                                    ≈ 원본과 비슷
                                     복원 오차 = 작음 → ✅ 정상

불량 이미지 입력:
  [불량 이미지] → Encoder → Latent → Decoder → [복원 이미지]
                                                    ≠ 원본과 다름!
                                     복원 오차 = 큼  → 🚨 이상!

이유: 모델은 정상만 학습했으므로, 정상은 잘 복원하지만
      한번도 본 적 없는 불량 패턴은 제대로 복원하지 못함
```

---

### 6-3. Anomaly Detection vs Classification 비교

| 비교 항목 | Classification (분류) | Anomaly Detection (이상 탐지) |
|-----------|:---------------------:|:-----------------------------:|
| **학습 데이터** | 정상 + 불량 **모두 필요** | 정상 데이터**만** 필요 |
| **불량 데이터 요구량** | 수백~수천 장 | **0장** (필요 없음) |
| **새로운 불량 유형 대응** | 학습하지 않은 불량은 탐지 불가 | ✅ 학습하지 않은 불량도 탐지 가능 |
| **출력** | 클래스 라벨 (정상/불량 종류) | Anomaly Score (정상과의 거리) |
| **불량 위치 파악** | ❌ (분류만 가능) | ✅ 가능 (Heatmap으로 시각화) |
| **한계** | 불량 데이터 부족 시 성능 저하 | "정상의 다양한 변이"가 충분해야 함 |

### 적용 시 기대효과

- ✅ **불량 데이터 없이도 학습 가능**: 극심한 불균형 상황에서 유일한 해법
- ✅ **새로운 불량 유형도 탐지**: 학습 시 본 적 없는 불량도 "정상과 다르다"는 이유로 탐지
- ✅ **불량 위치 시각화 가능**: Heatmap으로 어디가 이상인지 시각적으로 표시
- ⚠️ 정상 데이터가 충분히 다양해야 함 (정상의 다양한 변이를 커버하지 못하면 오탐 발생)
- ⚠️ 불량의 종류(scratch, dent 등)를 구분하지는 못함 (정상 vs 비정상만 판단)

> 📌 **실무 Tip**: 라이브러리 [Anomalib](https://github.com/openvinotoolkit/anomalib)을 사용하면
> PatchCore, PaDiM 등 다양한 Anomaly Detection 모델을 손쉽게 실험할 수 있습니다.

## 7. 기법 비교 총정리

| 기법 | 레벨 | 접근 방법 | 불량 데이터 필요 | 구현 난이도 | 효과 |
|------|------|-----------|:---------------:|:-----------:|------|
| **Augmentation** | 데이터 | 소수 클래스 데이터를 변환/증식 | ✅ 필요 | ⭐⭐ | 데이터 다양성 증가 + 과적합 방지 |
| **Sampler** | 샘플링 | Batch 구성 시 소수 클래스 우선 선택 | ✅ 필요 | ⭐ | 원본 보존하며 균형 학습 |
| **Weighted CE** | 손실 함수 | Loss에 클래스별 가중치 부여 (정적) | ✅ 필요 | ⭐ | 가장 간단, 소수 클래스 오분류 패널티 |
| **Focal Loss** | 손실 함수 | 어려운 샘플에 자동 집중 (동적) | ✅ 필요 | ⭐⭐ | 가장 정교, 객체 탐지에 특히 유효 |
| **Anomaly Detection** | 패러다임 전환 | 정상만 학습, 정상과 다르면 불량 판단 | ❌ 불필요 | ⭐⭐⭐ | 불량 0장이어도 가능, 미지의 불량 탐지 |

## 8. 실무 적용 가이드

### 📌 불량 데이터 양에 따른 전략 선택

```
불량 데이터 양에 따른 의사결정:

  불량 수천 장 이상
      │
      └─→ Classification (①②③ 기법 조합)

  불량 수백 장
      │
      └─→ Augmentation + Sampler + Focal Loss (최대 보정)

  불량 수십 장
      │
      ├─→ 보정 기법으로 시도해 볼 수 있지만 성능 한계
      └─→ Anomaly Detection 전환 검토

  불량 10장 미만 또는 0장
      │
      └─→ Anomaly Detection (정상만 학습)  ← 유일한 현실적 선택
```

### 📌 상황별 추천 조합

```
▶ 시작 단계 (빠른 실험):
   Weighted CE  →  가장 간단하게 불균형 대응

▶ 이미지 분류 (중간 불균형):
   Augmentation + Sampler + Weighted CE  →  3가지 레벨 동시 적용

▶ 이미지 분류 (극심한 불균형):
   Augmentation + Sampler + Focal Loss  →  최대 효과 조합

▶ 객체 탐지:
   Augmentation + Focal Loss  →  배경 vs 객체 극심한 불균형에 효과적

▶ 불량 데이터가 거의 없는 경우 (불량 < 수십 장):
   Anomaly Detection (PatchCore 등)  →  정상만 학습, 불량 데이터 불필요
```

### 📌 핵심 원칙

1. **학습 데이터에만 적용**: 증강은 Train 세트에만! (Validation/Test에는 원본 사용)
2. **여러 기법을 조합**: 한 가지만 쓰기보다 데이터 레벨 + 손실 함수 레벨을 함께 적용
3. **올바른 평가 지표 사용**: 불균형 데이터에서는 **Accuracy 대신 F1-Score, AUROC** 활용
4. **실험으로 검증**: 정답은 없으므로 여러 조합을 실험하며 최적의 전략을 찾아야 함
5. **패러다임 전환도 고려**: 불량이 극소량이면 ①②③ 보정보다 **④ Anomaly Detection이 현실적**
