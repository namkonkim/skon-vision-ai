## 1. 라벨링 환경 구축 — Labelme

### 📌 Labelme란?

[Labelme](https://github.com/wkentaro/labelme)는 MIT에서 개발한 오픈소스 이미지 라벨링 도구입니다.

| 특징 | 설명 |
|------|------|
| **지원 형식** | Polygon, Bounding Box, Point, Line, Circle 등 다양한 Annotation 지원 |
| **출력 포맷** | JSON (이미지별 1개 파일 생성) |
| **호환성** | COCO, PASCAL VOC 등 주요 포맷으로 변환 가능 |
| **설치 방식** | pip 한 줄로 설치, GUI 제공 |

### 📌 설치 방법

```bash
# pip으로 설치
pip install labelme

# 실행
labelme
```

### 📌 기본 사용법

```
1. 메뉴 → Open Dir → 이미지가 있는 폴더 선택
2. 우측 라벨 목록에서 작업할 이미지 선택
3. 좌측 도구 모음에서 Annotation 도구 선택
   - Create Rectangle  → Bounding Box
   - Create Polygon     → Polygon (Segmentation)
   - Create Rotation Box → Oriented Bounding Box (회전 가능한 Labelme Fork 필요)
4. 객체 영역을 지정하고, 클래스 이름(Label) 입력
5. Ctrl+S 로 저장 → 이미지명.json 파일 생성
```

### 📌 Labelme JSON 출력 예시

```json
{
  "version": "5.0.1",
  "imagePath": "det_easy_1.jpg",
  "imageHeight": 480,
  "imageWidth": 640,
  "shapes": [
    {
      "label": "apple",
      "shape_type": "rectangle",
      "points": [[120, 80], [350, 320]]
    }
  ]
}
```

## 2. Vision AI Task별 라벨링 실습

각 Task별로 **5장의 이미지**를 준비했습니다.
쉬운 이미지(경계가 뚜렷)부터 어려운 이미지(경계가 모호)까지 난이도를 섞어 구성했습니다.

---

### 2-1. Classification (이미지 분류)

#### 라벨링 방식

Classification은 다른 Task와 달리 **이미지 자체에 Annotation을 그리지 않습니다.**
대신, **폴더 구조**로 클래스를 구분합니다.

```
images/
├── dog/           ← 클래스: "강아지"
│   ├── dog_easy_1.jpg
│   ├── dog_easy_2.jpg
│   └── dog_hard_1.jpg
└── cat/           ← 클래스: "고양이"
    ├── cat_easy_1.jpg
    └── cat_hard_1.jpg
```

> 📌 **PyTorch의 `ImageFolder`** 를 사용하면, 이 폴더 구조를 자동으로 읽어 라벨을 부여합니다.

#### 실습 이미지

| 파일명 | 난이도 | 설명 |
|--------|:------:|------|
| `dog/dog_easy_1.jpg` | 쉬움 | 밝고 선명한 강아지 사진 |
| `dog/dog_easy_2.jpg` | 쉬움 | 배경이 깔끔한 강아지 사진 |
| `cat/cat_easy_1.jpg` | 쉬움 | 배경이 단순한 고양이 사진 |
| `dog/dog_hard_1.jpg` | 어려움 | 어둡거나 가려진 강아지 사진 |
| `cat/cat_hard_1.jpg` | 어려움 | 배경과 구분이 어려운 고양이 사진 |

#### 실습 과제
1. 이미지를 보고 **올바른 폴더에 분류**하세요.
2. 어려운 이미지의 경우, **어떤 기준으로 분류해야 할지** 생각해 보세요.

---

### 2-2. Object Detection (바운딩 박스)

#### 라벨링 방식

대상 객체를 **직사각형(Bounding Box)**으로 감싸고, 클래스 이름을 지정합니다.

```
Labelme 설정:
  도구: Create Rectangle
  출력: shape_type = "rectangle"
  좌표: [[x_min, y_min], [x_max, y_max]]
```

#### 실습 이미지

| 파일명 | 난이도 | 설명 |
|--------|:------:|------|
| `det_easy_1.jpg` | 쉬움 | 단일 객체, 배경이 깨끗 |
| `det_easy_2.jpg` | 쉬움 | 여러 객체가 떨어져 있음 |
| `det_medium.jpg` | 보통 | 객체들이 가까이 있지만 겹치지 않음 |
| `det_hard_1.jpg` | 어려움 | 객체들이 겹쳐 있어 개별 bbox 설정이 어려움 |
| `det_hard_2.jpg` | 어려움 | 일부 객체가 이미지 경계에서 잘림 |

#### 실습 과제
1. Labelme에서 **Create Rectangle** 도구를 사용하여 모든 객체에 Bounding Box를 그리세요.
2. 겹치는 객체의 경우, 각 객체별로 **별도의 Box**를 그려야 합니다.
3. 이미지 경계에서 잘린 객체도 **보이는 부분만큼** Box를 그려야 합니다.

---

### 2-3. Segmentation (폴리곤 마스크)

#### 라벨링 방식

대상 객체의 **외곽선을 따라 폴리곤(다각형)**을 그려 픽셀 단위로 영역을 지정합니다.

```
Labelme 설정:
  도구: Create Polygon
  출력: shape_type = "polygon"
  좌표: [[x1,y1], [x2,y2], [x3,y3], ...]  (다각형 꼭짓점 목록)
```

#### 실습 이미지

| 파일명 | 난이도 | 설명 |
|--------|:------:|------|
| `seg_easy_1.jpg` | 쉬움 | 배경과 대비가 뚜렷, 단순한 형태 |
| `seg_easy_2.jpg` | 쉬움 | 배경과 색상 차이가 큰 객체 |
| `seg_medium.jpg` | 보통 | 복잡한 형태이지만 경계는 뚜렷 |
| `seg_hard_1.jpg` | 어려움 | 객체와 배경의 색상/질감이 비슷하여 경계가 모호 |
| `seg_hard_2.jpg` | 어려움 | 불규칙한 형태(나뭇가지 등)로 폴리곤 그리기 어려움 |

#### 실습 과제
1. Labelme에서 **Create Polygon** 도구를 사용하여 객체 외곽을 따라 폴리곤을 그리세요.
2. 복잡한 형태는 **포인트를 촘촘히** 찍어야 정확합니다.
3. 경계가 모호한 경우, **일관된 기준**을 정해서 그리세요 (예: "보이는 부분까지만").

---

### 2-4. Oriented Bounding Box (회전 바운딩 박스)

#### 라벨링 방식

일반 BBox와 달리, **객체의 기울기에 맞춰 회전된 직사각형**을 그립니다.
특히 항공/위성 영상에서 배, 차량, 비행기 등 방향이 있는 객체에 사용합니다.

```
OBB 좌표 형식:
  (x_center, y_center, width, height, angle)

  또는 4개 꼭짓점:
  [(x1,y1), (x2,y2), (x3,y3), (x4,y4)]
```

> 📌 Labelme 기본 버전에서는 **Create Polygon 도구로 4개 꼭짓점**을 찍어 OBB를 표현할 수 있습니다.
> 또는 OBB를 지원하는 확장 버전([roLabelImg](https://github.com/cgvict/roLabelImg) 등)을 사용할 수 있습니다.

#### 실습 이미지

| 파일명 | 난이도 | 설명 |
|--------|:------:|------|
| `obb_easy_1.jpg` | 쉬움 | 단일 회전 객체, 배경이 깔끔 |
| `obb_easy_2.jpg` | 쉬움 | 소수의 객체가 다른 각도로 배치 |
| `obb_medium.jpg` | 보통 | 여러 객체가 다양한 각도로 배치 |
| `obb_hard_1.jpg` | 어려움 | 많은 객체가 밀집, 각도 파악 어려움 |
| `obb_hard_2.jpg` | 어려움 | 객체들이 서로 교차/겹침 |

#### 실습 과제
1. 각 객체의 **기울어진 방향에 맞추어** 4개 꼭짓점(Polygon)을 찍으세요.
2. 일반 BBox와 OBB의 차이를 직접 체감해 보세요.
3. OBB가 필요한 이유: 기울어진 객체에 일반 BBox를 씌우면 **불필요한 배경 영역이 포함**됨.

```
일반 BBox (기울어진 객체에 적용 시):
  ┌─────────────────┐
  │    ╱──────╲      │  ← 빈 공간이 많음
  │  ╱  객  체  ╲    │     (배경이 포함됨)
  │╱──────────────╲  │
  └─────────────────┘

OBB (기울기에 맞춤):
    ╱──────────────╲
  ╱    객   체       ╲   ← 딱 맞는 박스!
  ╲──────────────────╱     (배경 최소화)
```

## 3. 성능을 높이는 라벨링 가이드라인

좋은 모델은 **좋은 데이터**에서 시작됩니다.
아무리 좋은 모델 구조와 학습 전략을 써도, **라벨링 품질이 낮으면 성능은 올라가지 않습니다.**

```
"Garbage In, Garbage Out"
   — 잘못된 라벨 → 잘못된 학습 → 잘못된 예측
```

---

### 3-1. 일관성 있는 라벨링 기준 수립

라벨링에서 가장 중요한 것은 **일관성(Consistency)**입니다.
여러 사람이 라벨링할 때, 각자 다른 기준으로 라벨링하면 모델이 혼란을 겪습니다.

#### 📌 라벨링 기준 문서(Labeling Specification)에 반드시 포함할 내용

| 항목 | 설명 | 예시 |
|------|------|------|
| **클래스 정의** | 각 클래스가 무엇을 의미하는지 명확히 정의 | "scratch" = 표면에 0.5mm 이상의 선형 흠집 |
| **경계 기준** | 객체의 경계를 어디까지 포함할지 규칙 정의 | "그림자는 포함하지 않음", "가려진 부분은 추정하여 그림" |
| **최소 크기** | 라벨링할 객체의 최소 크기 기준 | "10×10 픽셀 미만의 객체는 라벨링하지 않음" |
| **겹침 처리** | 객체가 겹칠 때 처리 방법 | "50% 이상 가려진 객체는 occluded 라벨 추가" |
| **경계 사례 (Edge Case)** | 판단이 어려운 사례에 대한 명확한 지침 | "정상/불량 구분이 모호하면 불량으로 분류" |
| **시각적 가이드** | Good/Bad 예시를 이미지로 제시 | ✅ 올바른 라벨 / ❌ 잘못된 라벨 비교 이미지 |

---

### 3-2. Task별 라벨링 품질 체크리스트

#### Object Detection (Bounding Box)

```
✅ Good Labeling:
  ┌────────┐
  │  🍎    │  ← 객체에 딱 맞는 Box
  └────────┘     (여백 최소화)

❌ Bad Labeling:
  ┌────────────────┐
  │                │  ← Box가 너무 큼
  │     🍎        │     (불필요한 배경 포함)
  │                │
  └────────────────┘

❌ Bad Labeling:
      ┌──────┐
      │ 🍎   │       ← Box가 너무 작음
      └──────┘          (객체 일부가 잘림)
```

| 체크 항목 | 기준 |
|-----------|------|
| **Tightness** | Box가 객체에 최대한 밀착되게 그렸는가? |
| **Completeness** | 모든 대상 객체에 Box를 그렸는가? (누락 없음) |
| **Class 정확성** | 올바른 클래스 이름을 부여했는가? |
| **잘린 객체** | 이미지 경계에서 잘린 객체도 보이는 부분만큼 Box를 그렸는가? |
| **겹침 처리** | 겹치는 객체를 각각 별도의 Box로 그렸는가? |

#### Segmentation (Polygon)

| 체크 항목 | 기준 |
|-----------|------|
| **경계 정밀도** | 폴리곤이 객체 외곽선을 정확히 따르는가? |
| **포인트 밀도** | 곡선 부분에 충분한 포인트를 찍었는가? |
| **배경 포함** | 폴리곤 안에 불필요한 배경이 포함되지 않았는가? |
| **구멍 처리** | 객체 안에 구멍(hole)이 있으면 별도 처리했는가? |

#### Oriented Bounding Box

| 체크 항목 | 기준 |
|-----------|------|
| **각도 정확성** | Box의 회전 각도가 객체의 기울기와 일치하는가? |
| **Tightness** | 회전된 Box가 객체에 밀착되었는가? |
| **일관된 방향** | 같은 종류의 객체에 대해 일관된 방향(head/tail)을 유지하는가? |

### 3-3. 라벨링 품질 관리 (QA/QC)

대규모 라벨링 프로젝트에서는 **체계적인 품질 관리**가 필수적입니다.

#### 📌 QA/QC 프로세스

```
Step 1: 파일럿 라벨링 (Pilot)
  ├── 소량(50~100장)의 샘플을 전체 라벨러가 동시에 라벨링
  ├── 결과를 비교하여 기준의 모호한 부분 발견
  └── 라벨링 가이드라인 보완

Step 2: 본 라벨링 (Production)
  ├── 보완된 가이드라인에 따라 대규모 라벨링 수행
  └── 라벨러별 할당량 배분

Step 3: 품질 검수 (Quality Check)
  ├── 전체 라벨링의 10~20%를 랜덤 샘플링하여 검수
  ├── 오류율이 기준 초과 시 해당 라벨러의 결과를 전수 검수
  └── 공통 오류 패턴을 가이드라인에 추가

Step 4: 교차 검증 (Cross Validation)
  ├── 동일 이미지를 2명 이상이 독립적으로 라벨링
  ├── 라벨 일치도(Inter-Annotator Agreement, IAA) 측정
  └── 불일치 항목은 전문가가 최종 판정
```

#### 📌 라벨 일치도 지표

| 지표 | 설명 | 기준 |
|------|------|------|
| **IoU (Intersection over Union)** | 두 라벨러의 Box/Mask 영역 겹침 비율 | IoU ≥ 0.75 이면 일치 |
| **Cohen's Kappa** | 분류 일치도 (우연 일치 보정) | κ ≥ 0.80 이면 높은 일치 |
| **오류율** | 전체 중 수정이 필요한 라벨 비율 | 5% 이하 유지 목표 |

### 3-4. 성능 향상을 위한 라벨링 Best Practices

| # | 원칙 | 설명 |
|---|------|------|
| 1 | **클래스를 최소화** | 불필요하게 세분화된 클래스보다, 명확히 구분 가능한 클래스로 설계 |
| 2 | **애매하면 기준을 강제** | "모호하면 ○○으로 라벨링한다" 같은 강제 규칙을 정해 일관성 확보 |
| 3 | **작은 객체도 빠짐없이** | 작은 객체를 누락하면 모델이 해당 크기를 무시하게 학습됨 |
| 4 | **잘린 객체도 라벨링** | 이미지 경계에 걸친 객체도 보이는 만큼 라벨링 (실제 추론 시에도 잘린 객체 등장) |
| 5 | **주기적 QA 실시** | 라벨링 초반에 발견하지 못한 오류가 전체에 전파되는 것 방지 |
| 6 | **Good/Bad 예시 공유** | 텍스트보다 **시각적 예시**가 기준 이해에 훨씬 효과적 |
| 7 | **버전 관리** | 라벨링 가이드라인을 변경할 때마다 버전을 기록하고, 이전 라벨과의 호환성 확인 |

## 4. 오토라벨링 (Auto Labeling)

수동 라벨링은 **시간과 비용이 매우 많이 소요**됩니다.
오토라벨링은 AI 모델을 활용하여 라벨링 작업을 자동화하거나 보조하는 기법입니다.

```
수동 라벨링:
  사람이 1장씩 직접 라벨링  →  시간 소요 ↑,  비용 ↑,  일관성 ↓

오토라벨링:
  AI가 자동으로 라벨 생성  →  시간 소요 ↓,  비용 ↓,  일관성 ↑
  (단, 사람이 검수하여 최종 확인)
```

### 4-1. SAM (Segment Anything Model) 활용

#### 📌 SAM이란?

Meta AI가 개발한 **범용 Segmentation 모델**로, 별도 학습 없이 어떤 이미지에서든 객체의 마스크를 자동 생성할 수 있습니다.

| 항목 | 설명 |
|------|------|
| **개발** | Meta AI (2023) |
| **특징** | Zero-shot Segmentation — 학습 없이 바로 사용 가능 |
| **입력** | 이미지 + 프롬프트 (Point, Box, 텍스트) |
| **출력** | 픽셀 단위의 Segmentation Mask |

#### 📌 SAM 활용 방식

```
방법 1: 자동 전체 마스크 생성 (Automatic Mask Generation)
  이미지를 넣으면 → 감지된 모든 객체의 마스크를 자동 생성
  → 사람이 클래스 이름만 부여하면 완료

방법 2: 포인트/박스 프롬프트 (Interactive)
  사람이 객체 위치를 점(Point)이나 Box로 지정
  → SAM이 해당 영역의 정밀한 마스크를 자동 생성
  → 수동 폴리곤 작업 대비 5~10배 빠름
```

#### 📌 SAM 기반 오토라벨링 워크플로우

```
Step 1: SAM 모델 로드
  │  from segment_anything import sam_model_registry, SamAutomaticMaskGenerator
  │  sam = sam_model_registry["vit_h"](checkpoint="sam_vit_h.pth")
  ▼
Step 2: 이미지에서 자동 마스크 생성
  │  mask_generator = SamAutomaticMaskGenerator(sam)
  │  masks = mask_generator.generate(image)
  ▼
Step 3: 생성된 마스크 확인 및 클래스 부여
  │  각 마스크에 대해 사람이 클래스 이름 지정
  │  또는 CLIP 등 모델로 자동 클래스 분류
  ▼
Step 4: Labelme JSON 또는 COCO 포맷으로 변환/저장
```

#### 📌 SAM 코드 예시

```python
from segment_anything import sam_model_registry, SamAutomaticMaskGenerator
import cv2

# 모델 로드
sam = sam_model_registry["vit_h"](checkpoint="sam_vit_h_4b8939.pth")
sam.to(device="cuda")

# 자동 마스크 생성
mask_generator = SamAutomaticMaskGenerator(
    model=sam,
    points_per_side=32,          # 포인트 그리드 밀도
    pred_iou_thresh=0.88,        # 예측 IoU 임계값
    stability_score_thresh=0.95, # 마스크 안정성 임계값
    min_mask_region_area=100     # 최소 마스크 면적 (px)
)

image = cv2.imread("target_image.jpg")
image = cv2.cvtColor(image, cv2.COLOR_BGR2RGB)

masks = mask_generator.generate(image)
# masks: 각 마스크별 segmentation, bbox, area, predicted_iou 등 포함

print(f"감지된 객체 수: {len(masks)}")
```

### 4-2. 모델 학습 → 추론 → 자동 라벨링 파이프라인

SAM은 범용 모델이라 **클래스를 자동 부여하지 못합니다.**
우리 도메인에 특화된 오토라벨링을 하려면, **소량의 수동 라벨링 → 모델 학습 → 대량 자동 라벨링** 순서로 진행합니다.

#### 📌 전체 파이프라인

```
┌─────────────────────────────────────────────────────────────────┐
│                    오토라벨링 파이프라인                          │
│                                                                 │
│  Phase 1: 소량 수동 라벨링 (Seed Labeling)                       │
│  ─────────────────────────────────────                          │
│    전체 데이터 중 소량(100~500장)을 수동으로 라벨링               │
│         │                                                       │
│         ▼                                                       │
│  Phase 2: 초기 모델 학습                                         │
│  ──────────────────────                                         │
│    수동 라벨링된 데이터로 간단한 모델(YOLOv8 등)을 학습           │
│         │                                                       │
│         ▼                                                       │
│  Phase 3: 라벨링 안 된 데이터에 추론                              │
│  ─────────────────────────────                                  │
│    학습된 모델로 나머지 라벨링 안 된 이미지를 추론                 │
│    각 예측 결과에 Confidence Score가 함께 출력됨                  │
│         │                                                       │
│         ▼                                                       │
│  Phase 4: Confidence 기반 자동 라벨링                            │
│  ─────────────────────────────────                              │
│    Confidence ≥ 임계값 (예: 0.90)  →  자동 라벨 확정  ✅          │
│    Confidence < 임계값              →  사람이 수동 검수  🔍       │
│         │                                                       │
│         ▼                                                       │
│  Phase 5: 반복 (Active Learning Loop)                           │
│  ────────────────────────────────                               │
│    수동 검수 결과를 학습 데이터에 추가 → 모델 재학습              │
│    → 더 높은 정확도로 다시 Phase 3~4 반복                        │
│    → 점점 수동 작업이 줄어듦                                     │
└─────────────────────────────────────────────────────────────────┘
```

#### 📌 Confidence 기반 필터링 코드 예시

```python
from ultralytics import YOLO
import json

# 학습된 모델 로드
model = YOLO("best.pt")

# 라벨링 안 된 이미지들에 대해 추론
results = model.predict(source="unlabeled_images/", save=False)

CONFIDENCE_THRESHOLD = 0.90  # 이 값 이상이면 자동 라벨로 확정

auto_labeled = []
need_review = []

for result in results:
    image_path = result.path
    high_conf_boxes = []

    for box in result.boxes:
        confidence = box.conf.item()
        cls = int(box.cls.item())
        xyxy = box.xyxy[0].tolist()  # [x1, y1, x2, y2]

        if confidence >= CONFIDENCE_THRESHOLD:
            high_conf_boxes.append({
                "class": model.names[cls],
                "confidence": round(confidence, 4),
                "bbox": xyxy
            })

    if high_conf_boxes:
        auto_labeled.append({
            "image": image_path,
            "labels": high_conf_boxes
        })
    else:
        need_review.append(image_path)

print(f"✅ 자동 라벨링 완료: {len(auto_labeled)}장")
print(f"🔍 수동 검수 필요:  {len(need_review)}장")
```

#### 📌 Confidence 임계값 설정 가이드

| 임계값 | 자동 라벨 비율 | 정확도 | 적합한 상황 |
|:------:|:--------------:|:------:|-------------|
| **0.95** | 낮음 (~30%) | 매우 높음 | 품질이 매우 중요한 경우 (의료, 자율주행) |
| **0.90** | 중간 (~50%) | 높음 | 일반적인 산업용 검사 |
| **0.80** | 높음 (~70%) | 보통 | 빠른 대량 라벨링이 필요한 초기 단계 |
| **0.70** | 매우 높음 (~85%) | 낮음 | 1차 필터링 후 전수 검수 예정인 경우 |

> 📌 **핵심**: 임계값이 높을수록 자동 라벨의 정확도는 올라가지만, 자동 처리되는 비율은 낮아집니다.
> 실무에서는 보통 **0.85~0.95** 범위에서 시작하여 결과를 보며 조정합니다.

## 📝 Chapter 12 핵심 정리

| 주제 | 핵심 내용 |
|------|-----------|
| **라벨링 도구** | Labelme — Rectangle(BBox), Polygon(Segmentation), 4점 Polygon(OBB) |
| **Classification** | 폴더 구조로 클래스 구분, Annotation 파일 없음 |
| **Object Detection** | 직사각형 BBox로 객체를 감쌈, Tightness가 핵심 |
| **Segmentation** | 폴리곤으로 객체 외곽선을 정밀하게 추적 |
| **OBB** | 객체 기울기에 맞춘 회전 BBox, 항공/위성 영상에 유용 |
| **라벨링 가이드** | 일관성이 가장 중요, 클래스 정의 + 경계 기준 + Edge Case 규칙 필수 |
| **QA/QC** | 파일럿 → 본 라벨링 → 검수 → 교차 검증 프로세스 |
| **오토라벨링 (SAM)** | Zero-shot Segmentation으로 마스크 자동 생성, 클래스는 수동 부여 |
| **오토라벨링 (학습 기반)** | 소량 수동 라벨링 → 모델 학습 → Confidence 높은 예측을 자동 라벨로 확정 |
